# CLAUDE.md

このファイルは、リポジトリ内のコードを扱う際に Claude Code (claude.ai/code) へ情報を提供します。

## このリポジトリについて

個人の Neovim 設定。Claude の役割はユーザーと相談しながら設定を育てていくこと。
現在は `init.lua` 単一ファイル構成。プラグインが増えて管理しにくくなったら `lua/plugins/` 等への分割を検討すること。分割した場合はこのファイルの構成説明も合わせて更新すること。

## 重要な制約

- `vim.g.mapleader` は `require("lazy")` より前に設定しなければならない（Neovim の制約。逆にすると leader キーが効かない）

## コーディング規約

- 各設定・プラグインには「なぜこの設定が必要か」「なぜこのプラグインを選んだか」を日本語コメントで記載する
- プラグインのコメントブロックには役割・採用理由・必要な外部依存を含める

## プラグインの追加方法

`init.lua` 内の `require("lazy").setup({})` にスペックテーブルを追加する。

- 遅延読み込みには `cmd` / `ft` / `event` を使う
- キーマップは `keys` テーブルに書く
- `lazy-lock.json` はプラグインのバージョンを固定している。プラグインの追加・更新時は合わせて更新されることを確認する
- 追加後は `README.md` も更新する（キーマップの追記、外部 CLI が必要な場合はインストール方法の記載）

## Git コミット規約

コミットメッセージは以下のプレフィックスを使う：

- `feat:` 新しいプラグインや設定の追加
- `fix:` 動作の修正
- `docs:` README.md 等のドキュメント更新

## 動作確認

設定を変更・追加したあとは Neovim を再起動して動作確認を促すこと。
